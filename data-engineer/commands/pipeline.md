# /data:pipeline

Design a data pipeline using pipeline-design skill.

## Input Required

1. **Source(s)** - Where does data come from?
2. **Target** - Where should it land?
3. **Frequency** - Real-time, hourly, daily?
4. **Volume** - Approximate data size?

## Output

- Architecture diagram (ASCII)
- Technology recommendations
- Error handling strategy
- Monitoring approach
