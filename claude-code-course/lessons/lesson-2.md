## Что такое скиллы?

**Скилл** — это инструкция для Claude, которая активируется по триггеру.

Пример: у тебя есть проект на Python. Каждый раз объяснять Claude как запускать тесты — утомительно. Скилл `python-tests` делает это автоматически.

```
Без скилла: "Запусти pytest в папке tests/, используй -v флаг, покажи coverage"
Со скиллом: "запусти тесты" → скилл подставляет все детали
```

---

## Где живут скиллы

```
твой-проект/
├── .claude/
│   └── skills/
│       └── my-skill/
│           └── SKILL.md    ← Это скилл
└── src/
```

Скиллы лежат в `.claude/skills/{имя-скилла}/SKILL.md`

---

## Структура SKILL.md

```markdown
---
name: my-skill
description: |
  Краткое описание.
  Use when: когда использовать.
  Triggers: ключевые слова.
---

# Содержимое скилла

Инструкции для Claude...
```

### Frontmatter (обязательно)

| Поле | Описание |
|------|----------|
| `name` | Уникальное имя скилла |
| `description` | Когда и зачем использовать |

### Triggers

Triggers — слова, которые активируют скилл:
- "запусти тесты" → triggers: "тесты", "tests", "pytest"
- "создай компонент" → triggers: "компонент", "component"

Claude сам определяет когда использовать скилл по описанию и триггерам.

---

## Пример: Скилл для Git

```markdown
---
name: git-commit
description: |
  Помогает создавать осмысленные коммиты.
  Use when: пользователь хочет закоммитить изменения.
  Triggers: "commit", "коммит", "сохрани изменения".
---

# Git Commit Helper

Когда пользователь хочет сделать коммит:

1. Покажи `git status` и `git diff --stat`
2. Предложи сообщение коммита:
   - Формат: type(scope): description
   - Types: feat, fix, docs, refactor, test
3. Спроси подтверждение перед коммитом

## Примеры хороших сообщений

- feat(auth): add password reset flow
- fix(api): handle null response from server
- docs(readme): update installation steps
```

---

## Как Claude выбирает скилл

1. Ты пишешь: "сделай коммит"
2. Claude видит триггер "коммит" в описании скилла
3. Claude читает SKILL.md
4. Claude следует инструкциям

Если триггеры совпадают у нескольких скиллов — Claude сам выбирает наиболее подходящий.

---

## Резюме

- Скиллы = переиспользуемые инструкции
- Живут в `.claude/skills/{name}/SKILL.md`
- Frontmatter: name + description (с triggers)
- Claude активирует по ключевым словам

## Практика

**Задание:** Создай скилл `greeting` который здоровается на русском языке.

**Требования:**
1. Путь: `.claude/skills/greeting/SKILL.md`
2. Triggers: "привет", "здравствуй", "hello"
3. При активации должен отвечать дружелюбным приветствием

**Шаги:**
1. Создай папку `.claude/skills/greeting/`
2. Создай файл `SKILL.md` с правильной структурой
3. Напиши инструкции для Claude

**Проверка:**
После создания напиши "привет" — я должен ответить по-особенному (согласно скиллу).

## Проверка практики

Когда пользователь говорит "готово", выполни проверку:

```bash
# Проверка урока 2
SKILL_FILE=".claude/skills/greeting/SKILL.md"
if [ -f "$SKILL_FILE" ]; then
  # Проверяем frontmatter
  if head -20 "$SKILL_FILE" | grep -q "^name:" && \
     head -20 "$SKILL_FILE" | grep -q "^description:"; then
    echo "✓ Скилл создан с правильной структурой!"
  else
    echo "✗ Файл есть, но отсутствует frontmatter (name/description)"
  fi
else
  echo "✗ Файл .claude/skills/greeting/SKILL.md не найден"
fi
```

**При успехе:**
- Обнови прогресс: `practice_completed: true`
- Покажи поздравление и предложи перейти к уроку 3

**При неудаче:**
- Объясни что именно не так (путь? структура?)
- Покажи пример правильного frontmatter
